## LVM磁盘分区（以1T SSD为例）

- 先创建1GB的EFI分区，挂载到`/boot/efi`
- 手动创建一个 **EXT4 分区**，挂载到 `/boot` (2GB)
- 将剩下的空间创建一个分区，类型选为 **"Physical Volume for LVM"**
- 在这个 PV 上创建一个 **卷组 (VG)**，命名为 `system_vg`
- 在 `system_vg` 里创建 **逻辑卷 (LV)**：
  - `root_lv` -> 挂载到 `/` (100GB)
  - `home_lv` -> 挂载到 `/home` （750GB）
  - `swap_lv` -> 设为 swap (64GB)
  - **保留未分配空间**: ~50GB（作为缓冲，想给谁给谁）

```bash
# 1. 将磁盘初始化为物理卷 (PV)
sudo pvcreate /dev/sdb

# 查看结果
sudo pvs

# 2. 创建名为 vg_data 的卷组
sudo vgcreate vg_data /dev/sdb

# 查看卷组剩余空间
sudo vgs

# 3. 创建一个名为 lv_storage 的逻辑卷，使用全部剩余空间
# -n 指定名称，-l +100%FREE 表示分配所有空间
sudo lvcreate -l +100%FREE -n lv_storage vg_data

# 或者指定具体大小：sudo lvcreate -L 50G -n lv_storage vg_data

# 4. 格式化为 ext4 文件系统
sudo mkfs.ext4 /dev/vg_data/lv_storage 

# 5. 持久化到 fstab (实现开机自动挂载)
# 获取逻辑卷的 UUID
sudo blkid /dev/vg_data/lv_storage

# 创建挂载目录
sudo mkdir -p /data

# 备份 fstab 以防万一
sudo cp /etc/fstab /etc/fstab.bak

# 将挂载信息写入 fstab (请替换下方 UUID)
# 格式：[UUID] [挂载点] [文件系统类型] [参数] [dump] [pass]
echo "UUID=a1b2c3d4-e5f6-7890... /data ext4 defaults 0 2" | sudo tee -a /etc/fstab

# 卸载当前已挂载的（如果有），然后尝试通过 fstab 自动挂载
sudo mount -a

# 检查是否挂载成功
df -h | grep /data

sudo chown :dev_group /data
sudo chmod 2775 /data
```

## 基础深度学习环境安装

### 1. 前提条件

安装 Ubuntu 24.04 系统时，**不要勾选**“为图形硬件安装第三方软件”。进入系统后，确保联网。

### 2. 安装驱动和CUDA包

这一步会将 NVIDIA 官方驱动和 CUDA 13.1 全部装入系统。

```bash
# 1. 下载并配置优先权文件 (Pin file)
wget https://developer.download.nvidia.com/compute/cuda/repos/ubuntu2404/x86_64/cuda-ubuntu2404.pin
sudo mv cuda-ubuntu2404.pin /etc/apt/preferences.d/cuda-repository-pin-600

# 2. 下载 4GB 的本地仓库安装包 (deb local)
wget https://developer.download.nvidia.com/compute/cuda/13.1.0/local_installers/cuda-repo-ubuntu2404-13-1-local_13.1.0-590.44.01-1_amd64.deb

# 3. 安装仓库包并添加密钥
sudo dpkg -i cuda-repo-ubuntu2404-13-1-local_13.1.0-590.44.01-1_amd64.deb
sudo cp /var/cuda-repo-ubuntu2404-13-1-local/cuda-*-keyring.gpg /usr/share/keyrings/

# 4. 更新源并执行“全家桶”安装
sudo apt-get update
# 注意：这里使用 'cuda' 而不是 'cuda-toolkit'，确保驱动一并安装
sudo apt-get -y install cuda

# 5. 必须重启系统以加载内核驱动
sudo reboot
```

### 3.安装cuDNN包

```bash
wget https://developer.download.nvidia.com/compute/cudnn/9.17.1/local_installers/cudnn-local-repo-ubuntu2404-9.17.1_1.0-1_amd64.deb
sudo dpkg -i cudnn-local-repo-ubuntu2404-9.17.1_1.0-1_amd64.deb
sudo cp /var/cudnn-local-repo-ubuntu2404-9.17.1/cudnn-*-keyring.gpg /usr/share/keyrings/
sudo apt-get update
sudo apt-get -y install cudnn-cuda-12
```

### 4. 配置全局环境变量

重启后，将 CUDA 路径写入 `~/.bashrc`，让系统能找到 `nvcc`。

```bash
echo 'export PATH=/usr/local/cuda-13.1/bin${PATH:+:${PATH}}' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=/usr/local/cuda-13.1/lib64${LD_LIBRARY_PATH:+:${LD_LIBRARY_PATH}}' >> ~/.bashrc
source ~/.bashrc
```

直接配置整个系统的cuda路径，对所有的用户同时生效。

```bash
sudo sh -c "echo 'export PATH=/usr/local/cuda-13.1/bin:\$PATH' > /etc/profile.d/cuda.sh" 
sudo sh -c "echo 'export LD_LIBRARY_PATH=/usr/local/cuda-13.1/lib64:\$LD_LIBRARY_PATH' >> /etc/profile.d/cuda.sh" 
source /etc/profile.d/cuda.sh
```

## 构建共享的Anaconda基座

### 1. 核心思路：建立“公共家园”

我们不在某个具体用户的目录下安装，而是在 `/home` 根目录下创建一个伪用户目录（例如 `/home/anaconda`），并以此作为共享基座。

### 2. 标准操作流程

#### 第一步：创建目录并分配给组

```bash
# 在 /home 下创建安装点
sudo mkdir /home/anaconda

# 创建共享用户组
sudo groupadd ai_group

# 将自己先加入到共享用户组
sudo usermod -aG ai_group {your_username}

# 将所有权交给共享组 ai_group
sudo chgrp -R ai_group /home/anaconda

# 赋予 SGID 权限，确保组员创建的环境目录属性正确
sudo chmod 2775 /home/anaconda

# 剥夺组员对 main 目录的写权限，只保留读和执行
sudo chmod -R 755 /home/anaconda/main

# 但要确保 pkgs 目录对 ai_group 是可写的（为了共享缓存）
sudo chmod -R 775 /home/anaconda/main/pkgs
sudo chmod -R g+s /home/anaconda/main/pkgs
```

#### 第二步：安装 Anaconda 到该位置

下载 Anaconda/Miniconda 安装包后，运行安装程序：

- 当询问安装路径时，手动输入：`/home/anaconda/main`。
- **注意**：在安装结束问是否运行 `conda init` 时，选 **no**。我们不需要它去污染 root 用户的环境。

#### 第三步：配置全局访问权限 (关键)

为了让所有用户都能直接使用 `conda` 命令，修改全局配置文件 `/etc/profile.d/conda.sh`：

```Bash
# 创建一个新的 shell 初始化脚本
sudo nano /etc/profile.d/conda.sh
```

写入以下内容：

```bash
# 激活公共 conda 路径
export CONDA_PATH="/home/anaconda/main"
if [ -f "$CONDA_PATH/etc/profile.d/conda.sh" ]; then
    . "$CONDA_PATH/etc/profile.d/conda.sh"
fi
```

保存退出。这样，**任何用户登录系统后，`conda` 命令就是立即可用的**。

### 3. 如何优雅地解决权限冲突？

#### **原则：禁止普通用户修改 Base 环境**

- Base 环境（`/home/anaconda/main`）应该只有管理员有写权限。
- 普通用户只能**查看**和**使用**。

#### **策略：重定向用户环境到各自的家目录**

通过配置全局的 `.condarc`，强制让用户把新创建的环境保存在自己的 `/home/user/` 下。这样既利用了 SSD 的速度，又隔离了权限。

修改（或创建）全局配置文件 `/home/anaconda/main/.condarc`：

```yaml
# 只设置这个，让大家共享 SSD 上的包缓存，提高速度
pkgs_dirs:
  - /home/anaconda/main/pkgs
  - ~/.conda/pkgs

envs_dirs:
  - ~/.conda/envs
  - /home/anaconda/main/envs
```

### 4.配置conda channel及pip channel

```bash
# 1. 移除现有的所有 channels（可选，为了确保环境干净）
conda config --remove-key channels

# 2. 按照优先级“由低到高”反向添加，或者直接用 add 命令
# 我们先加低优先级的 defaults
conda config --add channels defaults

# 3. 再加高优先级的 tsinghua 和 conda-forge
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/conda-forge/

conda config --set show_channel_urls yes
```

## DDP训练及压力测试

### 1.随机生成矩阵

```python
import os
import torch
import torch.distributed as dist
import torch.multiprocessing as mp

def setup(rank, world_size):
    os.environ['MASTER_ADDR'] = 'localhost'
    os.environ['MASTER_PORT'] = '12355'
    # 使用 NCCL 后端，这是验证宿主机 CUDA/cuDNN 环境的最佳方式
    dist.init_process_group("nccl", rank=rank, world_size=world_size)

def cleanup():
    dist.destroy_process_group()

def ddp_test(rank, world_size):
    setup(rank, world_size)
    
    # 确保每个进程锁定一张不同的显卡
    torch.cuda.set_device(rank)
    device = torch.device(f"cuda:{rank}")
    
    # 执行随机任务：生成大矩阵并求和
    print(f"[Rank {rank}] 正在 GPU {torch.cuda.get_device_name(rank)} 上启动随机任务...")
    data = torch.randn(5000, 5000, device=device)
    result = data.matmul(data.t()).sum()
    
    # DDP 核心步骤：多卡规约求和
    dist.all_reduce(result, op=dist.ReduceOp.SUM)
    
    if rank == 0:
        print(f"--- 测试完成！多卡通信正常，DDP 规约结果已同步 ---")
    
    cleanup()

if __name__ == "__main__":
    # 自动检测可用显卡数量
    world_size = torch.cuda.device_count()
    if world_size < 2:
        print("提示：检测到单卡环境，将以单进程模式验证 NCCL 初始化...")
        world_size = 1
    
    mp.spawn(ddp_test, args=(world_size,), nprocs=world_size, join=True)
```

### 3.多节点多卡DDP测试

#### 训练主文件

```python
import os
import time
import argparse
import torch
import torch.distributed as dist
from torch.nn.parallel import DistributedDataParallel as DDP

def setup():
    dist.init_process_group(backend="nccl")
    local_rank = int(os.environ["LOCAL_RANK"])
    global_rank = int(os.environ["RANK"])
    torch.cuda.set_device(local_rank)
    return local_rank, global_rank

def run_stress_test(config):
    local_rank, global_rank = setup()
    device = torch.device(f"cuda:{local_rank}")

    # 根据输入的 config 构建深层大矩阵网络
    layers = []
    for _ in range(config.num_layers):
        layers.append(torch.nn.Linear(config.hidden_dim, config.hidden_dim))
        layers.append(torch.nn.ReLU())

    model = torch.nn.Sequential(*layers).to(device)
    model = DDP(model, device_ids=[local_rank])

    optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
    criterion = torch.nn.MSELoss()

    if global_rank == 0:
        print("="*50)
        print(f"🚀 3090 多机多卡压力测试启动")
        print(f"   维度 (Hidden Dim): {config.hidden_dim}")
        print(f"   层数 (Num Layers): {config.num_layers}")
        print(f"   批量 (Batch Size): {config.batch_size}")
        print(f"   总进程数 (World Size): {dist.get_world_size()}")
        print("="*50)

    step = 0
    while True:
        start_time = time.time()

        # 生成大规模随机数据
        inputs = torch.randn(config.batch_size, config.hidden_dim).to(device)
        targets = torch.randn(config.batch_size, config.hidden_dim).to(device)

        # 前向 + 后向 (触发 All-Reduce 通信)
        outputs = model(inputs)
        loss = criterion(outputs, targets)

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

        iter_time = time.time() - start_time

        if step % 10 == 0 and global_rank == 0:
            print(f"Step {step:4d} | Iter Time: {iter_time:.3f}s | Loss: {loss.item():.4f}")

        step += 1

if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    # 压力调节参数
    parser.add_argument("--hidden_dim", type=int, default=8192, help="矩阵维度 (控制显存和计算量)")
    parser.add_argument("--num_layers", type=int, default=10, help="层数 (控制计算深度和梯度大小)")
    parser.add_argument("--batch_size", type=int, default=64, help="每张卡的 Batch Size")

    args = parser.parse_args()
    run_stress_test(args)
```

## 数据盘权限分配

### 第一阶段：准备用户组与用户

首先，创建一个专门的 AI 团队用户组，并将相关人员加入其中。

```bash

# 2. 将现有用户添加到组（替换 user1, user2 为实际用户名）
sudo usermod -aG ai_group user1
sudo usermod -aG ai_group user2

# 3. 刷新组权限（用户需重新登录才能生效，或者执行下面命令临时生效）
newgrp ai_group
```

------

### 第二阶段：设置 /data 目录基础所有权

假设你的 15TB HDD 已经挂载到了 `/data`。

```bash
# 1. 创建挂载点（如果还没创建）
sudo mkdir -p /data

# 2. 修改目录的所有者为 root，但所属组设为 ai_group
sudo chown root:ai_group /data

# 3. 设置基础权限：所有者(rwx)、组(rwx)、其他人(rx)
sudo chmod 775 /data
```

------

### 第三阶段：应用核心 SGID 权限

**SGID (Set Group ID)** 是多用户协作的神器。设置后，任何人在 `/data` 下创建的新文件或子目录，其**所属组**会自动继承父目录的 `ai_group`，而不是创建者的私有组。

```bash
# 设置 SGID 权限
# 2775 中的 '2' 代表 SGID 位
sudo chmod 2775 /data

# 如果目录下已经有旧文件，递归应用 SGID 和 组写权限
sudo chmod -R g+s /data
sudo chmod -R g+w /data
```

------

### 第四阶段：细分目录权限

为了防止核心数据集被误删，建议对子目录进行差异化授权：

| 目录             | 权限建议                               | 命令                            |
| ---------------- | -------------------------------------- | ------------------------------- |
| `/data/datasets` | **公共数据集**：仅管理员可写，组员只读 | `sudo chmod 755 /data/datasets` |
| `/data/models`   | **公共权重**：组内共享，均可读写       | `sudo chmod 2775 /data/models`  |
| `/data/code`     | **项目区**：各用户建立自己的子目录     | `mkdir /data/code`              |

### 第五阶段：umask 设置

**修改 `/etc/profile`**： 使用 root 权限打开文件：

```bash
sudo nano /etc/profile
```

在文件末尾添加：

```bash
# Set default umask for all users to allow group write access
umask 002
```
